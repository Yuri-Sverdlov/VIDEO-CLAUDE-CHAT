## Часть 1. Фразы для чата с архитектором

Это **не** slash-команды Cursor — фразы, которые ты говоришь агенту в чате.

### 1. Начать новую сессию (восстановить контекст)

```
Прочитай CONTEXT.md, PROJECT_LOG.md и tasks/REPORT.md.
Кратко: где мы остановились и что делать дальше?
```

**Порядок чтения:**
1. `CONTEXT.md` — суть и текущий фокус
2. `PROJECT_LOG.md` — история сессий
3. `tasks/REPORT.md` — отчёт кодера
4. `tasks/TASK.md` — активное задание

### 2. Закрыть сессию (сохранить для следующего раза)

```
Закрой сессию: обнови PROJECT_LOG.md и CONTEXT.md.
Активное задание оставь/создай в tasks/TASK.md.
```

### 3. Выдать задание кодеру

```
Подготовь задание для кодера в tasks/TASK.md: [кратко что сделать].
Дай мне текст для вставки в терминал.
```

**Текст для терминала (стандарт):**
```
Прочитай CONTEXT.md, затем AGENTS.md, затем tasks/TASK.md — и выполни задание.
По завершении заполни tasks/REPORT.md.
```

### 4. Проверить и принять работу кодера

```
Проверь tasks/REPORT.md по критериям из tasks/TASK.md.
Принято или дай правки.
```

Если не принято:
```
Не принято. Обнови tasks/TASK.md — только конкретные правки, без переделки всего.
```

### 5. Сменить модель / роль исполнителя

```
С этого шага [описание задачи] делай ты в чате, кодера не привлекай.
```
```
Это задание только для кодера в терминале — подготовь TASK.md, сам код не пиши.
```
```
Для следующего задания кодеру рекомендую модель [название] — учти в TASK.md.
```

| Кто | Где | Когда |
|-----|-----|-------|
| Архитектор | Чат | план, ТЗ, проверка, логи |
| Кодер | Терминал | код, запуск, тесты, git (если в TASK) |

Git commit/push — почти всегда кодер в терминале.

### 6. Git-чекпоинт (сохранить на GitHub)

```
Подготовь git-задание для кодера: закоммитить [что] и запушить на GitHub.
```

### 7. Обновить «куда идём» без нового кода

```
Обнови CONTEXT.md: текущий фокус — [одно предложение].
Добавь в backlog tasks/backlog/YYYY-MM-DD_название.md.
```

### 8. Статус «где мы сейчас»

```
Дай статус проекта: TASK, REPORT, последний коммит, что блокирует.
```

### 9. Откатить / только посмотреть

```
Только анализ, ничего не меняй в файлах.
```
```
Откати [файл/изменение] к состоянию последнего коммита.
```
```
Покажи diff между локальным и GitHub — без правок.
```

### 10. Следующий шаг пайплайна (стейдж-гейт)

```
Активируй стадию [A/B/C]: подготовь TASK.md для кодера.
Напомни критерии приёмки.
```

**VIDEO-CLAUDE-CHAT:**
- **A** — `python champion_movie.py --images-only` (сначала картинки)
- **B** — видео (только после «ок» на кадры)
- **C** — сборка ffmpeg

Кодер не переходит к следующей стадии без явного задания в `TASK.md`.

### Файлы двухагентной схемы

| Файл | Кто пишет | Зачем |
|------|-----------|-------|
| `CONTEXT.md` | архитектор | память + текущий фокус |
| `PROJECT_LOG.md` | архитектор | история сессий |
| `tasks/TASK.md` | архитектор | одно активное задание |
| `tasks/REPORT.md` | кодер | отчёт по заданию |
| `tasks/done/` | архитектор | архив принятых работ |
| `AGENTS.md` | архитектор | правила для кодера |

### Ритуал на каждый день

**Начало:** `Прочитай CONTEXT.md, PROJECT_LOG.md, tasks/REPORT.md. Где мы?`

**Работа:** `Подготовь TASK.md для кодера: [задача].`

**Конец:** `Закрой сессию: PROJECT_LOG + CONTEXT.`

## Часть 2. Slash-команды Cursor (`/команда`)

Набираются в поле ввода чата или CLI. Набери **`/`** — Cursor покажет все доступные команды (зависит от версии и режима).

**Важно:** набор команд **разный** в чате IDE и в **Cursor CLI** (`cursor agent` в терминале).

Документация CLI: https://cursor.com/docs/cli/reference/slash-commands

---

### 10 базовых slash-команд

| # | Команда | Где | Зачем |
|---|---------|-----|-------|
| 1 | `/help` | CLI | Справка. Пример: `/help model` |
| 2 | `/summarize` или `/compress` | Чат IDE + CLI | Сжать длинный диалог, освободить контекст |
| 3 | `/resume` | CLI | Продолжить прошлый чат по папке проекта |
| 4 | `/new-chat` | CLI | Начать чистый чат |
| 5 | `/model` | CLI | Список или смена модели: `/model`, `/model gpt-5` |
| 6 | `/plan` | CLI | Режим планирования — сначала план, потом код |
| 7 | `/ask` | CLI | Только чтение/анализ, без правок файлов |
| 8 | `/create-rule` | Чат IDE | Создать правило в `.cursor/rules/` |
| 9 | `/commands` | CLI | Создать/редактировать свои slash-команды |
| 10 | `/worktree` | Чат IDE | Агент в изолированном git worktree |

---

### Сохранение сессии / восстановление

| Действие | Как в Cursor |
|----------|--------------|
| Сжать текущий чат | `/summarize` или `/compress` |
| Продолжить прошлый чат (CLI) | `/resume` + папка проекта |
| Новый чат, та же папка | `/new-chat` |
| Подтянуть прошлую переписку (IDE) | **`@Past Chats`** (не slash — выбрать чат) |
| Надёжное сохранение контекста проекта | `CONTEXT.md` + `PROJECT_LOG.md` (файлы в репо) |

### Смена модели

| Где | Как |
|-----|-----|
| Чат IDE | Выпадающий список модели над полем ввода |
| Cursor CLI | `/model` — список; `/model имя-модели` — выбрать |
| Сравнить модели | `/best-of-n` (чат IDE) |

### Планирование без кода

| Где | Как |
|-----|-----|
| CLI | `/plan` |
| Чат IDE | **Shift+Tab** — Plan Mode |

---

### Кастомные slash-команды

Файл `.cursor/commands/имя.md` → команда `/имя`.

```
.cursor/commands/
├── session-start.md    → /session-start
├── session-close.md    → /session-close
└── task-coder.md       → /task-coder
```

Создать через CLI: `/commands`.

Глобальные команды: `~/.cursor/commands/`.

Документация: https://cursor.com/docs/agent/chat/commands

---

### Мини-шпаргалка slash-команд

```
Новая сессия в CLI     →  /resume
Длинный чат, тормозит  →  /summarize
Сменить модель (CLI)   →  /model
Только спросить        →  /ask
Сначала план           →  /plan (CLI)  или  Shift+Tab (IDE)
Свои команды           →  /commands
Справка (CLI)          →  /help
Все доступные          →  просто  /
```

### Дополнительные команды (полезно знать)

| Команда | Где | Зачем |

| `/multitask` | Чат IDE | Параллельные подзадачи |
| `/loop` | Чат IDE | Повторять промпт по расписанию |
